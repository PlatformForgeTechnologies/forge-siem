Why Forge SIEM

Multi-cloud, one platform

Connect AWS, Azure, GCP, or Linode. Each integration activates its own collection pipeline — CloudTrail, GuardDuty, VPC Flow Logs, Activity Log. Nothing runs for infrastructure you don't have.

Capability-based activation

Pick your infrastructure during onboarding. Only the relevant collectors and detection rules activate. Add new platforms later from settings — no reinstall, no downtime, no surprise bills.

Your data, your cloud

Raw logs never leave your cloud account. Forge runs detection rules in memory and stores only alert metadata. Built for GDPR, HIPAA, and data residency requirements without any extra configuration.

Real-time Sigma detection

Bundled detection rules mapped to MITRE ATT&CK. Rules fire on decoded events before they're stored — sub-second alert latency. Tune or extend with your own Sigma YAML.

Analyst-ready dashboard

Alert triage, case management, threat hunts, network and database activity monitoring, vulnerability tracking — all in one place. No pivot to a separate tool to answer "what happened?"

BYOC data plane

Keep raw events in your own cloud. Deploy the Forge data plane into your Kubernetes cluster via Helm. An outbound-only relay tunnel connects back to Forge — no inbound firewall rules or public endpoints required. Raw logs never leave your environment. Charts at forge-siem-charts.

Behavioural detection (UEBA)

Forge baselines normal process execution, authentication, and event-rate patterns per host. Deviations from that baseline fire anomaly alerts without needing a matching Sigma rule — catching novel attacks that signature rules miss.

Multi-channel alerting

Route alerts to Slack, email, generic webhooks, PagerDuty, or Microsoft Teams. Each destination has its own severity and rule filters. Use the PagerDuty integration for on-call escalation and webhooks for Jira, n8n, or any custom automation.
