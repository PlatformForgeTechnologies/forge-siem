Features

Everything a security team needs

Built for operators who want Wazuh-level functionality with Kubernetes-native ergonomics.

Agent mTLS transport

Lightweight Go agent runs on Linux, macOS, and Windows. Enrolls via PKI, communicates over TLS 1.3 with mutual auth. Log tailing, FIM, process monitoring, package inventory, and active response.

Real-time detection

31+ Sigma-format detection rules across Linux auth, process, FIM, network, Kubernetes, and Windows auth. Rules engine evaluates decoded events in-stream and fires alerts to Redis within milliseconds.

ClickHouse event backend

MergeTree tables for events, alerts, and raw events with TTL-based lifecycle. Sub-second search across millions of events using field:value syntax with timeline bucketing.

Analyst dashboard

React dashboard with 14 pages: Overview, Alerts, Agents, Rules, Vulnerabilities, Logs, Network, Cases, Hunts, Dashboards, Reports, and Settings. Live SSE alert stream, MITRE ATT&CK overlays, alert drawer with remediation guidance.

Active response

HMAC-signed response commands queued to Redis, polled by the agent. Full RabbitMQ fanout for notifications and webhooks. Slack alerts for high/critical events. Response log with job tracking.

Vulnerability scanning

NVD CVE catalog with 2000+ CVEs polled every 6h. Per-agent package inventory matching. Trivy Operator integration for container image scanning — VulnerabilityReport CRDs persisted to PostgreSQL.

Compliance tagging

Every alert is automatically mapped from MITRE ATT&CK to CIS Controls v8 and NIST 800-53 Rev 5. Compliance breakdown visible in alert details and exportable per-tenant.

Helm + ArgoCD

Full Helm chart with self-hosted and BYO dependency modes. ArgoCD app-of-apps with PreSync bootstrap jobs. Pi/dev overlay with fully self-contained in-cluster dependencies. KEDA autoscaling on stream lag.

RBAC + audit log

Email/password login with HttpOnly session cookies. PBKDF2-SHA256 password hashing. Four roles: analyst, responder, admin, viewer. Every admin action written to the audit log.

Installation

Up in minutes on any cluster

01

Add the Helm repo

shell
helm repo add forge-siem \
  https://platformforgetechnologies.github.io/forge-siem-platform
helm repo update
02

Install on a Pi / single-node cluster

Use the bundled values-pi.yaml overlay for a fully self-contained install. All dependencies (ClickHouse, PostgreSQL, Redis, RabbitMQ) run in-cluster.

shell
helm install siem forge-siem/siem \
  --namespace siem --create-namespace \
  -f https://raw.githubusercontent.com/PlatformForgeTechnologies/\
forge-siem-platform/main/k8s-apps/siem/values-pi.yaml
03

Install an agent

Download the forge-siem-agent binary and enroll it against your cluster's ingest endpoint. The agent handles mTLS automatically.

shell — Linux / macOS
curl -fsSL https://github.com/PlatformForgeTechnologies/\
forge-siem-agent/releases/latest/download/install.sh \
  | INGEST_HOST=siem-ingest.siem.svc.cluster.local \
    ENROLL_TOKEN=<your-token> sh
04

Open the dashboard

Access the dashboard at the hostname configured in your Ingress (default: siem.platformforgetechnologies.com). Log in with the bootstrap credentials from your values file.

shell — port-forward for local access
kubectl port-forward -n siem svc/siem-dashboard 8080:80
Full documentation including BYO-dependency configuration, ArgoCD setup, and TLS configuration is in the docs.
